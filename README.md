# SQL_Data_Warehouse_Project
# 📊 Data Warehouse & Analytics Project

## Welcome 👋

Welcome to my Data Warehouse and Analytics Project.

I created this project to practice and showcase the skills I've been learning in Data Engineering and Data Analytics. The goal was to build a complete data solution from the ground up — starting with raw data, transforming it into a structured data warehouse, and then using SQL to generate meaningful business insights.

Rather than focusing only on writing queries, this project follows the full analytics workflow that many organizations use in the real world. It covers data cleaning, integration, modeling, and reporting, giving me hands-on experience with the different stages of a modern data platform.

## 🎯 Project Goal

The main objective of this project is to build a centralized data warehouse that brings together information from multiple business systems and transforms it into valuable insights for decision-making.

By the end of the project, the data becomes easier to understand, analyze, and use for business reporting.

# 🏗️ Data Engineering Phase

## What I Built

For the data engineering portion of this project, I designed and developed a data warehouse using SQL Server.

The warehouse combines data from different business systems into a single source of truth that can be used for reporting and analytics.

### Data Sources

The project uses data from two different business systems:

- ERP (Enterprise Resource Planning)
- CRM (Customer Relationship Management)

Both datasets were provided as CSV files.

### Data Cleaning

Before loading the data into the warehouse, I performed several data quality checks and transformations to improve reliability.

Some of the tasks included:

- Handling missing values
- Removing inconsistencies
- Standardizing formats
- Cleaning duplicate or invalid records

### Data Integration

After cleaning the data, I combined information from the different source systems into a unified data model designed specifically for analytical queries.

This makes reporting faster, simpler, and more consistent.

### Project Scope

For this project, I focused only on the latest available data.

Historical tracking and slowly changing dimensions were intentionally left out so I could focus on building a strong foundation in data warehousing concepts.

# 📈 Analytics & Reporting Phase

Once the data warehouse was completed, I used SQL to explore the data and generate business insights.

The analysis focuses on three key areas:

## Customer Behavior

Understanding how customers interact with the business.

Questions explored include:

- Who are the most valuable customers?
- What purchasing patterns exist?
- Which customer groups generate the most revenue?

## Product Performance

Analyzing how products perform across different metrics.

Examples include:

- Best-selling products
- Revenue contribution by product
- Product performance trends

## Sales Trends

Exploring how sales change over time.

This includes:

- Monthly sales analysis
- Revenue growth trends
- Seasonal patterns
- Overall business performance
